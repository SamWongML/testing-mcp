**Handoff notes (P9 тАФ 2026-07-24):**

- **Shape.** P9 is the agent-facing layer: the deterministic InsomniaтЖТIR scaffolder (`atp import`),
  the golden-master parity helper, the five MCP workflow prompts, `MIGRATION.md`, and the finalized
  `CLAUDE.md` surface reference. No engine/schema/store changes тАФ purely additive on top of P8.

- **Files:**
  - `packages/cli/src/import.ts` тАФ `importInsomnia(yamlText) тЖТ { files, mapping }` (pure transform)
    + `writeImport(yamlPath, root)` (the CLI command layer: read тЖТ transform тЖТ write drafts +
    `MIGRATION.md`) + `renderMigration(mapping, source)`. Parses **Insomnia v5 YAML**
    (`type: collection.insomnia.rest/5.0`) via the new `yaml` dep on `@atp/cli`.
  - `packages/cli/src/golden.ts` тАФ `goldenAssertions(baseline) тЖТ DeclarativeAssertion[]` (status +
    per-scalar-key shape) + `renderAssertions()` (TS source to paste). ┬з19 step 4.
  - `packages/cli/src/__fixtures__/petstore.insomnia.yaml` тАФ a representative v5 export exercising
    requestтЖТtest, folderтЖТsuite, env, bearer auth, and a `{% response %}` response-ref tag.
  - `packages/cli/src/index.ts` тАФ the `atp import <insomnia.yaml>` dispatch case + usage.
  - `packages/mcp-server/src/prompts/index.ts` тАФ `registerPrompts(server)`: the five prompts, each
    an argument schema + an instruction template rendered over the real tool/CLI surface. Wired
    unconditionally into `buildMcpServer` (prompts are pure guidance, so they exist on the sync
    surface too тАФ not gated on `ctx.db`).

- **┬з13.1 mapping the importer implements** (the *clean, deterministic* parts):
  request тЖТ a `defineTest` step ┬╖ folder/`children` тЖТ a `defineSuite` with an inline node per child ┬╖
  `environments.data` тЖТ `tests/_shared/env/<domain>.ts` (`defineEnv`) ┬╖ bearer `authentication` тЖТ
  `tests/_shared/auth/<domain>.ts` (`bearerAuth`) + `request.authRef` ┬╖ `{{ _.var }}` тЖТ `{{env.var}}`
  (auth token тЖТ `{{secrets.<VAR>}}`) ┬╖ JSON body text тЖТ an object literal. Names slug to
  `<domain>.<name>` ids; the returned `mapping` (Insomnia meta-id тЖТ IR id, `<suiteId>#<nodeId>` for
  suite nodes) becomes the `MIGRATION.md` table.

- **The "messy remainder" is deliberately deferred to the agent** (the plan's design): an Insomnia
  `{% response ... %}` chaining tag becomes an `__TODO_CHAIN__` placeholder plus a `// TODO(migrate)`
  comment pointing at the `import_insomnia_collection` prompt. The scaffold **compiles** as-is; the
  agent wires the real `extract` + `{{nodes.X.var}}` + `needs`. Every generated request also carries a
  placeholder `assert: [{ path: "status", op: "lt", value: 500 }]` for the golden-master step to refine.

- **`regenerate_reports` is prompt-only, by decision (confirmed with the user).** `get_report`
  already re-renders any stored run into any format, so the prompt just drives `list_runs` тЖТ
  `get_report {runId, format}` over historical runs тАФ no new tool, keeping the surface minimal.
  Documented as such in the prompt itself.

  37 pg-gated skips) / compile all green. No DB needed for any P9 test.
